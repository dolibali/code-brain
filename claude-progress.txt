Session: 2026-04-24

What was implemented
- Refactored the biggest architectural hotspots without changing the thin-service behavior.
- Split the CLI into modular command files under `src/cli/commands/` with shared helpers in `src/cli/helpers.ts`.
  `src/cli.ts` is now just a thin compatibility export to `src/cli/index.ts`.
- Split page storage responsibilities:
  - `src/pages/repository.ts` now acts as the coordinator
  - `src/pages/page-filesystem.ts` owns Markdown file walking/parsing/writes
  - `src/pages/page-indexer.ts` owns SQLite index synchronization
  - `src/pages/page-project.ts` owns project resolution helpers
  - `src/pages/types.ts` defines DTOs and the `PageService` interface
- Split MCP registration:
  - `src/mcp/server.ts` now only creates and serves the MCP server
  - `src/mcp/tools/*.ts` register individual tools
  - `src/mcp/helpers.ts` centralizes result/error handling
- Split storage migration logic:
  - `src/storage/index-db.ts` now only opens the DB and delegates to `runMigrations`
  - `src/storage/migrator.ts` introduces a lightweight migration runner with `schema_migrations`
  - `src/storage/migrations/00x-*.ts` holds explicit migration steps
- Added lightweight interface/type layers for `pages`, `links`, and `search`, and updated `open-service.ts` to expose interface-shaped service ports instead of concrete class types.
