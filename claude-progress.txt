What was implemented
- Upgraded Code Brain from a repo-local dev CLI into a release-ready package shape:
  `private: false`, publish metadata, `LICENSE`, `exports`, `files`, `prepack`, and a real `codebrain` bin that points at built JS.
- Added `codebrain init` so a fresh user can bootstrap config, repo directories, and the index without hand-writing YAML first.
- Added a search-side OpenAI-compatible integration path for real LLM query expansion and reranking.
- Added optional embedding support with a local SQLite-backed `page_embeddings` index and best-effort embedding refresh on `put_page` and `reindex`.
- Kept the service boundary thin: no write-side LLM judgment, no automatic memory extraction, and no hidden branch rejection.
- Tightened MCP semantics:
  better tool descriptions, structured error payloads, and search strategy metadata to help models understand the tool intent.
- Added companion docs for agent usage:
  `docs/BRAIN_SYNC_RECIPE.md` plus integration notes for Claude Code, Codex, Cursor, and Gemini CLI.

Important notes
- Search is now async internally because LLM expansion/reranking and embedding lookup can be remote operations.
- LLM is still only used on the search path.
- Embedding remains optional and defaults to off.
- The package still uses Node's built-in `node:sqlite`, so experimental warnings may appear depending on the Node version.
